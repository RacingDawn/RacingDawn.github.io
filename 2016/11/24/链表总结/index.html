Linked List 题目汇总

  1. 遍历链表的相加：Add Two Numbers I II

    • I：将两链表的对应结点的值相加得到一个新的链表，需要设置一个carry来表示对应结点相加时是否产生进位

    • II：与I类似，不过这里的链表是正序的，并且题目要求不能改变原来链表的顺序来实现加法，所以需要先得到两个链表的长度，然后将对应位相加按正序相加，由于递归的应用会使链表又从尾部开始往头部方向返回，因此可以再将进位加至前一个结点

  2. 对链表执行删除某结点操作：

    • Delete Node in a Linked List：删除某个指定结点，除尾结点外

    • Remove Linked List Elements：删除值等于某给定值的所有结点，设置两个ListNode变量，分别表示遍历链表时的当前结点和前一结点

    • Remove Duplicates from Sorted List I II

      • I：对一个已排好序的链表，删除里面重复的结点

      • II：对一个已排好序的链表，删除里面重复的结点，只留非重复的结点

        思路：设置两个指针，一个指向重复结点的前一个，另一个指向重复结点的最后一个，删除这两个指针间的结点

    • Remove Nth Node From End of List：删除倒数第N个结点

      方法一：遍历两次链表，第一次确定链表长度，然后删除第length-K个结点

      方法二：设置两个指针，一开始都指向哑结点，然后将某个指针后移k个位置，，然后整体向后移动这两个节点，直到某个指针走到最后一个结点的时候，此时另一指针所指向的位置就是倒数第k+1个结点的位置

  3. 反转链表：

    • Reverse Linked List I) II

      • I：普通的反转链表

        迭代：

        递归：

        思路：n1 → … → nk-1 → nk → nk+1 ← … ← nm，nk+1到nm为已经反转的，而目前的结点为nk，因此要使nk+1的下一结点为nk，则需要nk.next.next = nk;

      • II：反转m～n之间的结点

        思路：通过设置两个指针找到m的前一个结点和n的后一个结点，然后将m～n之间的结点运用I的方法反转后，再与这两个结点相连

    • Rotate List：将链表向右平移k个位置

      思路：k=k%length，然后类似之前双指针

    • Reorder List：给定一个LinkedList L: L0→L1→…→Ln-1→Ln,将它重新排序为: L0→Ln→L1→Ln-1→L2→Ln-2→…

      思路：设置两个指针先找到链表的中间位置，然后将后半段反转（可用迭代，递归，栈），最后将前半段和反转后的后半段合并，类似的有Palindrome Linked List题目，也是找到half，然后将后半段反转，再比较

    • Swap Nodes in Pairs

      思路：通过设置一个dummy，找到其结对反转的一般性

  4. 有圈链表：Linked List Cycle I II

    • I：简单的检测链表是否有圈

      • 方法一：遍历链表，通过hashtable存储每个结点

      • 方法二：通过双指针，一个fast一个slow，如果有圈指针所指位置会重合

      • 方法三：将遍历过的结点的next指向head，如果有圈则总会遇到遍历过的结点其next为head

    • II：检测链表是否有圈，并且返回圈的起始结点

      • 方法一：运用I中的方法一很容易解决，但是会使用额外的空间

      • 方法二：类似于I中的方法二，先找到圈中的某个点

        此时slow距离head有（A+B）步，fast有2（A+B）步，A：圈中起始点begin距head的步数，B：slow在圈中所走的步数（即slow距begin的距离+数个圈的长度），且有（A+B）+kN=2（A+B），N为圈的长度

        所以A=k*N-B，那么当slow在圈中再走A步的话，head也走A步，它们刚好相遇

小结
