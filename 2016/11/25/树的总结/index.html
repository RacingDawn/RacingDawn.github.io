树的路径

  1. 从根到叶子结点的路径

    Binary Tree Paths

    思路：递归的思想；到叶子结点终止，分为左右孩子路径递归

  2. 路径和（递归时sum-root.val，迭代时使用Stack，DFS）

    • Path Sum：计算是否有根到叶子结点路径和等于给定值

      思路：①类似于求Binary Tree Paths，求出所有的路径的和，然后看是否包含有给定值②直接递归求解，每次递归时新的值=给定值-root值

    • Path Sum II：求出所有根到叶子结点路径和等于给定值的路径

      思路：递归和迭代（迭代通过stack）

    • Path Sum III：求出所有路径和等于给定值的路径数（该路径不需要从根到叶子，只要满足从上至下即可）

      思路：①对每个结点进行此结点、左、右结点的归并，找寻符合要求的路径②通过hashmap保留每个结点的前缀路径和，若从根到此结点的sum=前缀路径和+指定值，则result+1

    • Sum of Left Leaves所有左叶子结点的和

      思路：直接递归求解，考虑root.left.left为空的结点的左孩子

    • Sum Root to Leaf Numbers：root-to-leaf路径1->2->3代表数字123，求所有路径代表的数字之和

      思路：递归的将和相加

树的深度

  1. Minimum Depth of Binary Tree：求树的最小深度

    思路：对于非空结点，若左结点为空则深度为右子树的最小深度加1（同理右结点为空时），当左右都不为空时取左右子树深度小的

  2. Maximum Depth of Binary Tree：求树的最大深度

    思路：①类似上面的最小深度的递归②DFS或者BFS来求解，应用孩子结点的深度为父亲结点深度加1即可

LCA（树中两结点的高度最低的共同祖先）

  1. Lowest Common Ancestor of a Binary Tree

    思路：①递归找寻,left != null时表示在左子树中找到了p(q),则下一步只要看q(p)是否在右子树中,若在右子树中则返回root,否则说明q(p)也在左子树中,则当前的left为所求点

    ②迭代,保留root至p(q)的路径或者建立一个map,key为结点,value为父亲结点,再找寻p和q的LCA

  2. Lowest Common Ancestor of a Binary Search Tree

    思路:①当成普通的二叉树,那么就是上题②应用二叉查找树的性质:左结点的值<根结点的值<右结点的值

树的性质

  1. 判断两树是否一致

    Same Tree

    思路：直接对两树一起DFS，判断对应节点是否相等

  2. TreeLinkNode

    Populating Next Right Pointers in Each Node II：将每层节点的next指向后一节点

    思路：因为每层可视作为一个链表，所以可在每层开始设置一个哑结点

  3. 二叉树扁平化成链表

    Flatten Binary Tree to Linked List

    思路：链表顺序为preOrder顺序，所以只需将root.left接在root.right的位置上，将原先的root.right接在left subtree的”最右边”（即左子树preOrder时的最后一个节点）

    或者是DFS的递归实现，在递归返回时连接链表

树的分类

  1. 对称树：左右节点值对称相等

    Symmetric Tree

    思路：将左右子树分开，一边为sample，另一边为mirror，root也可一分为二，考察sample.value是否等于mirror.value

  2. 完全二叉树：只有最下面的两层结点度能够小于2，并且最下面一层的结点都集中在该层最左边的若干位置的二叉树

    Count Complete Tree Nodes：计算完全二叉树结点的个数

    思路：对于完全二叉树，左子树的高总是等于右子树高度或者加1

    当leftHeight=rightHeight时说明左子树是一棵完全二叉树，此时countNumber(root)=2^leftHeight+countNumber(root.right)

当leftHeight>rightHeight时说明右子树是一棵完全二叉树，此时countNumber(root)=2^rightHeight+countNumber(root.left)

    加法运算符优先于移位运算符；求2的次幂使用移位运算比调用pow快得多
